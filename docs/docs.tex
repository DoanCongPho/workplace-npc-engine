\textbf{Summary.}
This report describes a fully working prototype of an AI Co-Worker Engine for
Edtronaut's job simulation platform. The engine powers two AI co-workers from
the Gucci Group HRM simulation -- Group CEO (Lorenzo Bertelli) and Group CHRO --
as NPC-style agents with distinct personalities, persistent emotional memory, and
context-aware knowledge retrieval. The implementation uses \textbf{LangGraph},
\textbf{FastAPI}, \textbf{FAISS\,+\,Ollama}, and \textbf{OpenAI gpt-4o}, with a
two-tier rapport system and an LLM-based safety guard. A React/TypeScript frontend
completes the demo.
\end{mdframed}

% ---- 1.3 State management ----------------------------------
\subsection{State Management -- How Turn 1 Affects Turn 5}

The engine maintains a \textbf{RapportState} (score, momentum) per session.
Two tiers of update run each turn.

\vspace{0.4em}
\noindent\textbf{Tier 1 -- Rule-based, synchronous ($<$1\,ms)}

\begin{center}
\begin{tabular}{@{}lll@{}}
\toprule
\textbf{Signal} & \textbf{Calculation} & \textbf{Meaning} \\
\midrule
\texttt{length\_ratio} & $\min(\text{user words}/\text{NPC words},\;2)$ & Engagement depth \\
\texttt{politeness}   & overlap with 9 polite phrases & Respectful tone \\
\texttt{vocab\_mirror}& user/NPC token intersection ratio & Domain attunement \\
\texttt{domain\_vocab}& persona keyword hits / 5 & Strategic substance \\
\texttt{frustration}  & 1.0 if message $<$ 5 words & Drop-off signal \\
\bottomrule
\end{tabular}
\end{center}

Score update (with exponential momentum):
\[
  \delta = \sum_i w_i \cdot s_i
  \qquad
  \mu_t = 0.7\,\mu_{t-1} + 0.3\,\delta
  \qquad
  \text{score}_t = \operatorname{clamp}\!\bigl(\text{score}_{t-1} + \delta + 0.2\,\mu_t,\;0,\;100\bigr)
\]

The momentum term $\mu$ gives the system memory: a learner who annoys the CEO in
Turn~1 ($\delta = -6$) carries negative momentum into Turn~5 even if Turns~2--4 are
average. The tier re-evaluates every turn, so length and temperature adjust
automatically.

\vspace{0.4em}
\noindent\textbf{Tier 2 -- LLM judge, async, fire-and-forget (zero added latency)}

After the NPC replies, \texttt{gpt-4o-mini} scores four semantic axes
(politeness, domain depth, strategic substance, frustration) via structured output.
The weighted delta is applied to rapport \emph{for the next turn}. This catches
what Tier-1 misses: a message can use polite words but be strategically hollow,
and the judge detects the difference.

% ---- 2.2 Tool use ------------------------------------------
\subsection{Tool Use -- RAG Pipeline}

The \texttt{retrieve\_context} node gives every NPC access to a persona-specific
knowledge base without hard-coding facts into the system prompt.

\begin{enumerate}[leftmargin=*,noitemsep]
  \item \textbf{Index build (once per persona):} Knowledge JSON documents are
        embedded via Ollama \texttt{nomic-embed-text} (local, free), L2-normalised,
        and stored in a FAISS \texttt{IndexFlatIP}. Inner product on L2-normalised
        vectors equals cosine similarity.
  \item \textbf{Query-time retrieval:} The user message is embedded; top-3 documents
        by cosine score are retrieved.
  \item \textbf{Threshold gate:} Only if top-1 cosine score $\geq 0.55$ are documents
        injected into the system prompt. Below threshold the slot stays empty,
        preventing off-topic content from polluting the prompt.
  \item \textbf{Pre-filter:} Messages shorter than 4 words skip embedding entirely.
\end{enumerate}

\noindent\textit{Calibration:} domain-relevant queries score 0.69--0.85;
off-topic queries score 0.45--0.52.

\subsection{Latency vs Quality Trade-offs}

\begin{center}
\begin{tabularx}{0.96\linewidth}{@{}lXl@{}}
\toprule
\textbf{Mechanism} & \textbf{What it does} & \textbf{Saving} \\
\midrule
Tier-2 async judge &
  gpt-4o-mini rapport update fires after reply is sent &
  0\,ms added \\
Dynamic \texttt{max\_tokens} &
  Frustrated CEO cap = 50 tok; Engaged = 400 tok &
  Up to 8$\times$ faster \\
RAG threshold gate &
  Off-topic queries skip the 300\,ms Ollama embed call &
  $\sim$300\,ms \\
4-word pre-filter &
  Trivial messages skip embedding entirely &
  $\sim$300\,ms \\

  $\sim$1\,s (first-call only) \\
Safety gpt-4o-mini &

  Cost saving \\
\bottomrule
\end{tabularx}
\end{center}

The dominant latency is the main \texttt{gpt-4o} call (1.5--2\,s). The dynamic
token budget ensures that most turns (Neutral / Guarded) are already
sub-2\,s even without streaming.

\section{Part 3 -- The Director Layer (Supervisor Agent)}

\subsection{Design}

The Director is a deterministic supervisor that runs at every turn
\emph{before} the prompt is built. It is invisible to the learner.
When triggered, it injects a coaching cue into the system prompt so the
NPC delivers the hint naturally in its own voice.

A 3-turn cooldown prevents over-coaching. The last-hint turn is
persisted in \texttt{ConversationStore} so the cooldown survives across
HTTP requests.

\subsection{Three Detection Signals}

\begin{enumerate}[leftmargin=*,itemsep=4pt]
  \item \textbf{Semantic loop} -- If the cosine similarity between each
        pair of the last 3 user messages exceeds 0.85
        (via \texttt{all-MiniLM-L6-v2}), the learner is rephrasing the
        same idea without advancing.

  \item \textbf{Progress stall} -- After 5+ turns without any
        proposal-language keyword (\textit{plan, recommend, suggest,
        strategy, framework, initiative, approach}), the conversation
        has not moved toward an actionable output.

  \item \textbf{Domain drift} -- If rapport drops below 40 and the
        last 3 messages contain zero keywords from the persona's domain
        vocabulary, the learner has disconnected from the simulation
        context.
\end{enumerate}

\subsection{Hint Injection into NPC Response}

\begin{lstlisting}[language=Python,
    caption={Director hint woven into system prompt (build\_prompt node)}]
# After normal system text is assembled:
if state.get("director_hint"):
    system_text += (
        "\n\n[INTERNAL COACHING CUE -- do not quote this note directly, "
        "but let it subtly shape your response to help the learner progress]\n"
        + state["director_hint"]
    )
\end{lstlisting}

Because the cue is inside the system prompt, the main LLM reads it as a
private instruction and pivots the conversation naturally. A Frustrated
CEO says \emph{``Come back when you have a specific proposal,''} while
an Engaged CEO builds a bridge question.

\subsection{NPCAgent -- Core Class}

The exact signature from the assignment specification:

\begin{lstlisting}[language=Python,
    caption={NPCAgent -- pipeline and public interface}]
class NPCAgent:
    """Persona-agnostic NPC engine.
    Adding a new co-worker = new PersonaConfig only, no engine edits.
    """

    def __init__(self, persona_id: str = "ceo"):
        self.persona = PERSONAS[persona_id]   # pure config dataclass
        self.graph   = self._build_graph()    # LangGraph DAG

    def _build_graph(self):
        g = StateGraph(NPCState)
        # Snake pipeline: safety -> RAG -> director -> prompt -> LLM -> rapport
        g.add_node("safety_check",     self._safety_check_node)  # async, gpt-4o-mini
        g.add_node("retrieve_context", _retrieve_context_node)   # FAISS + Ollama
        g.add_node("director",         _director_node)           # 3 signals
        g.add_node("build_prompt",     _build_prompt_node)       # injects hints
        g.add_node("llm_call",         self._llm_node)           # gpt-4o
        g.add_node("update_state",     _update_state_node)       # Tier-1 rapport
        g.add_edge(START, "safety_check")
        g.add_edge("safety_check",     "retrieve_context")
        g.add_edge("retrieve_context", "director")
        g.add_edge("director",         "build_prompt")
        g.add_edge("build_prompt",     "llm_call")
        g.add_edge("llm_call",         "update_state")
        g.add_edge("update_state",     END)
        return g.compile()

    async def chat(
        self,
        persona_id:   str,
        user_message: str,
        session_id:   str,
    ) -> tuple[str, StateUpdate, dict]:
        """
        Returns:
            assistant_message  -- NPC reply (in-character, tier-aware)
            state_update       -- rapport_score, momentum, emotional_state,
                                  turn_count, director_hint
            safety_flags       -- category, phase, reasoning, rag_top_score
        """
        ...
        result = await self.graph.ainvoke(initial_state)
        # Tier-2 semantic judge fires async -- 0 ms added latency
        asyncio.create_task(self._tier2_update(...))
        return result["assistant_message"], state_update, result["safety_flags"]
\end{lstlisting}

\subsection{Safety Guard -- LLM-Based, In-Character}

The safety guard uses a two-phase approach so responses are always
in-character rather than canned refusal strings.

\begin{lstlisting}[language=Python,
    caption={Safety check -- semantic classifier + system-prompt injection}]
async def run_safety_check(user_message, persona, llm) -> dict:
    # Phase 1: keyword pre-filter (< 1 ms, zero false-positives)
    if _is_obvious_jailbreak(user_message):
        return {
            "safety_category": "jailbreak",
            "safety_phase": "keyword",
            "safety_injection": build_safety_injection("jailbreak", persona)
        }

    # Phase 2: gpt-4o-mini structured-output classifier (~250 ms)
    label = await _classify_message(user_message, persona, llm)
    # label.category: "safe" | "jailbreak" | "nda_probe" | "off_topic"

    return {
        "safety_category": label.category,
        "safety_phase": "llm",
        "safety_reasoning": label.reasoning,
        "safety_injection": build_safety_injection(label.category, persona)
    }
\end{lstlisting}

The injection (e.g.,
``\textit{[CONFIDENTIALITY] The learner has asked about information you
cannot share...}'')
is prepended to the system prompt. The main LLM generates a fully
in-character, tier-aware response. A Frustrated CEO gives one terse
sentence; an Engaged CHRO declines warmly and pivots to what she can
discuss.
